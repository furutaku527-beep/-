# カチカチくん（小役カウンター）アプリ 要件調査・実装プラン

実機のパチスロ用小役カウンター「勝ち勝ちくん」（通称カチカチくん、ガイドワークス製）と
同等の仕様を持つアプリを、本リポジトリ（ピカピカスロットPWA）に追加するための調査結果とプランです。

---

## 1. 実機仕様の調査結果

「勝ち勝ちくん」シリーズ（勝ち勝ちくんLED／クリア等）の公表情報・利用者解説から整理した仕様。

### 1.1 ハードウェア構成

| 部品 | 実機仕様 |
|---|---|
| カウントボタン ×4 | **白・赤・緑・黄** の4色。各ボタンに小役を自由に割り当ててカウント（例: ぶどう=白、チェリー=赤） |
| 黒 ■／▲ ボタン | **総回転数（ゲーム数）の入力**用。桁送り・数値増加で総ゲーム数をセット |
| 紫 ■ ボタン | **モード切替**（カウント表示 ⇔ ÷確率表示）。合算計算の起点にも使用 |
| 液晶ディスプレイ | 4カウンター値＋ゲーム数を同時表示。LEDモデルは**上下2段のWディスプレイ** |
| LED | ボタン押下時に発光してカウント受付をフィードバック（ミスカウント防止） |
| 電源 | ボタン電池。**オートパワーオフ**あり、電源OFFでもデータ保持 |

### 1.2 操作仕様

- **カウント**: 小役が出たら対応する色ボタンを押す → カウント+1、枠が点滅（LEDモデルは発光）、「カチッ」というクリック感
- **減算（ミス修正）**: ▲ボタンを押しながら対象の色ボタンを押すと −1
- **総回転数入力**: 黒の■／▲ボタンでキリの良い回転数（1000, 2000…）を入力。打ち始めゲーム数が入っていると「S」表示
- **確率算出**: 紫■ボタンで「÷モード」に切替 → 色ボタンを押すと `総回転数 ÷ カウント数` の分母（= 1/x.x の x）を表示
- **合算確率**: 紫■を押してから色ボタンを**2つ**押すと、2役合算の確率を表示
- **個別リセット**: リセットしたい色ボタンを**長押し**で 0 に戻る
- **全消去**: 裏面スイッチを「NO SAVE」に合わせて電源ON（アプリでは設定メニューの全リセットに相当）
- **シーン切替**: A / B / C の3モードを独立して持ち、通常時・ボーナス中・AT中などを別々にカウント。Wディスプレイ＋リンク機能の組み合わせで**最大9場面**に対応
- **リンク機能**: 上下段の連動により、役の重複率やART突入率などを1画面で算出可能

### 1.3 想定ユースケース

ジャグラー等ノーマルAタイプの設定推測が主用途。ぶどう確率・単独REG確率などを
リアルタイムに算出し、設定差のある小役の出現率から高低設定を見極める。

## 2. アプリ要件定義

### 2.1 機能要件（実機同等 = MUST）

- **F1. 4色カウントボタン**（白・赤・緑・黄）: タップで +1。押下時に視覚フィードバック（発光/点滅）＋クリック音＋バイブ（対応端末）
- **F2. 減算モード**: ▲を押しながら（アプリでは「−モード」トグル or ▲長押し中）色ボタンで −1。0未満にはならない
- **F3. 総回転数入力**: ▲▼ボタンによる増減（+1 / 長押しで高速リピート、+100/+1000のステップ切替）※実機の桁入力を踏襲しつつタッチ向けに最適化
- **F4. ÷（確率）モード**: 紫ボタンでカウント表示⇔確率表示を切替。確率は `1/x.x` 形式（分母を小数第1位まで）
- **F5. 合算計算**: 紫ボタン → 色ボタン2つの順で、選択2役の合算確率を表示
- **F6. 個別リセット**: 色ボタン長押し（800ms）で該当カウンターを0に（誤操作防止に確認バイブ＋点滅）
- **F7. 全データリセット**: メニューから全カウンター＋ゲーム数を消去（確認ダイアログあり）
- **F8. A/B/Cシーン切替**: 3シーンを独立保持。各シーンに4カウンター＋ゲーム数のセット
- **F9. Wディスプレイ**: 上下2段表示（例: 上段=通常時、下段=ボーナス中）。段ごとにアクティブ切替
- **F10. データ永続化**: localStorage（Zustand persist）に自動保存。アプリを閉じても復元（実機の「電源OFFでもデータ保持」に相当）
- **F11. 打ち始めゲーム数「S」表示**: スタートゲーム数が入力済みであることを示すインジケーター

### 2.2 拡張要件（アプリならでは = SHOULD、フェーズ2以降）

- **F12. ボタンへの役名ラベル割り当て**（ぶどう/チェリー/ベル等を自由設定）
- **F13. 設定推測アシスト**: 機種スペック表（まずはアイム相当＝本アプリREADMEの値）と実測確率を突き合わせて近似設定を表示
- **F14. ピカピカスロット連携**: シミュレーター側のプレイ中に成立役を自動カウントする「オート連動」モード（練習用）
- **F15. 履歴・グラフ**: カウント推移のスパークライン表示

### 2.3 非機能要件

- スマホ縦画面ファースト（片手の親指操作で全ボタンに届く配置）
- オフライン完全動作（既存PWA構成に同居）
- タップ反応 <50ms、誤タップ防止のボタンサイズ（最小44×44px以上、実際は大型ボタン）
- クリック音はWeb Audio APIで自前合成（既存 `src/audio/sfx.ts` の方式を踏襲、外部音源なし）
- 画面スリープ防止オプション（Wake Lock API、対応ブラウザのみ）

## 4. 技術設計

既存構成（Vite + React + TypeScript + Zustand + CSS Modules + Web Audio）をそのまま活用。

### 4.1 追加ファイル構成

```
src/
  counter/                    # カチカチくんのコアロジック（UI非依存・テスト可能）
    types.ts                  # CounterState, Scene, DisplayMode などの型
    counterLogic.ts           # 加算/減算/確率計算/合算/リセットの純関数
  state/
    counterStore.ts           # Zustandストア（persist、シーンA/B/C × 上下段）
  components/counter/
    CounterView.tsx           # 画面全体
    CounterDisplay.tsx        # 液晶風表示（上下段）
    ColorButton.tsx           # 4色ボタン（長押しリセット・減算対応）
    GameInput.tsx             # ▲▼ゲーム数入力
    SceneTabs.tsx             # A/B/C切替
    *.module.css
  audio/sfx.ts                # クリック音（カチッ）を追加
tests/
  counter.test.ts             # ロジックの単体テスト
```

### 4.2 状態モデル（骨子）

```ts
type SceneId = 'A' | 'B' | 'C';
type Row = 'upper' | 'lower';
type ColorKey = 'white' | 'red' | 'green' | 'yellow';

interface RowData {
  games: number;                    // 総回転数
  counts: Record<ColorKey, number>; // 4色カウント
  labels: Record<ColorKey, string>; // 役名ラベル（F12）
}
interface CounterState {
  scenes: Record<SceneId, Record<Row, RowData>>;
  activeScene: SceneId;
  activeRow: Row;
  displayMode: 'count' | 'ratio';   // 紫ボタンで切替
  combineSelection: ColorKey[];     // 合算計算の選択中ボタン（0〜2個）
  minusMode: boolean;               // ▲押下中の減算モード
}
```

確率計算: `denominator = games / count`（count=0 のときは `-.-` 表示）。
合算: `games / (countA + countB)`。すべて `counterLogic.ts` の純関数として実装しテストする。

### 4.3 既存アプリとの統合

- `App.tsx` にビュー切替（`slot` | `counter`）を追加。ヘッダーまたはメニューから遷移
- PWA manifest・Service Workerは既存のものに同居（追加設定不要）
- フェーズ2のオート連動（F14）は、`gameStore` の成立役イベントを `counterStore` に購読させる形で疎結合に実装

## 5. 実装フェーズ

| フェーズ | 内容 | 完了条件 |
|---|---|---|
| **1. コアロジック** | `counter/` の型・純関数＋Vitestテスト | 加減算・確率・合算・リセットのテストが全パス |
| **2. 基本UI** | 4色ボタン＋液晶表示＋ゲーム数入力＋÷モード（単一シーン） | 実機の基本操作フロー（カウント→G入力→確率表示）が再現できる |
| **3. 実機フル仕様** | 長押しリセット・減算・合算・A/B/Cシーン・Wディスプレイ・S表示・永続化 | 第2.1節のF1〜F11をすべて満たす |
| **4. 体験仕上げ** | クリック音・バイブ・LED発光演出・実機風液晶デザイン・スロット画面との相互遷移 | スマホ実機で片手操作が快適 |
| **5. 拡張（任意）** | ラベル設定・設定推測アシスト・シミュレーター連動・グラフ | F12〜F15 |

フェーズ1〜4で「実機と全く同じ仕様」の達成。フェーズ5はアプリならではの付加価値。

## 6. テスト計画

- `tests/counter.test.ts`: 純関数の単体テスト（境界: count=0の除算、0未満への減算、リセット、シーン独立性、永続化のシリアライズ）
- 手動確認: iPhone Safari（PWAスタンドアロン）で長押し・連打・バイブ・音を確認
- 既存の収束テスト（`npm test`）が引き続きパスすること
