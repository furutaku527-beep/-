# 日本株デイトレード手法 バックテストシステム

YouTuber「シンジ」氏の手法(noteで公開)が、実際に売買して利益が出るかを
検証するためのバックテストシステム。**証券会社には接続しない検証専用**。

- 対象: 国内株式(現物)のデイトレード(同一日内のエントリー/エグジット)
- データ: [J-Quants API](https://jpx-jquants.com/)(JPX公式・無料プラン)
- 検証手法: `strategy.md` に整理(エントリー/エグジット/損切り/銘柄スクリーニング)

> ⚠️ これは過去データでの検証ツールです。将来の利益を保証するものではなく、
> 投資判断は自己責任で行ってください。

## ⚙️ 構成

```
src/
  data/          J-Quants API からのデータ取得・ローカル保存
  strategy/      strategy.md の手法をルールベースで実装(売買シグナル判定)
  backtest/      過去データに戦略を適用するシミュレーションエンジン
  report/        勝率・PF・最大DD などの指標算出とグラフ出力
dashboard/       Streamlit による確認用Webダッシュボード(PC/iPhone対応)
scripts/         データ取得などの実行スクリプト
data/            取得データの保存先(.gitignore 対象・コミットされない)
tests/           テスト
strategy.md      検証する手法の定義(※手法のnote共有後に作成)
```

## 🔑 セットアップ

```bash
# 1. 依存パッケージ
pip install -r requirements.txt

# 2. J-Quants の APIキーを設定(V2)
cp .env.example .env
#   J-Quants ダッシュボードで発行した APIキーを .env の JQUANTS_API_KEY に記入
#   (.env は .gitignore でコミット対象外)

# 3. 接続テスト
python scripts/fetch_data.py --check
```

### J-Quants API V2 について(重要)

- **2026年6月1日に V1 API は廃止**され、**V2 API(APIキー方式)**に移行しました。
- 認証はメール/パスワードではなく、**ダッシュボードで発行する APIキー**(`x-api-key`)です。
- ベースURL `https://api.jquants.com/v2`、日足は `/equities/bars/daily`。

### J-Quants 無料プランについて

- 取得可能期間: **直近約2年**
- **約12週間のデータ遅延**あり(直近営業日が数か月前になるのは正常)
- **分足(ザラ場)データは無料プランでは提供されない見込み**
  → 本システムはまず **日足ベースの検証**から始めます。
  日中の値動きを使う検証(寄り→引け、ザラ場の高値/安値タッチ等)は、
  日足の OHLC で近似できる範囲と、分足が必須で近似できない範囲を分けて扱います。

## 📊 使い方

```bash
# 1. 日足データ取得(要 J-Quants 登録)
python scripts/fetch_data.py --codes 7203 6758 9984 --from 2023-01-01 --to 2024-12-31

# 1'. 大量銘柄をレート制限に優しくゆっくり取得(ローカルPC推奨・サンプル増に最適)
python scripts/fetch_data.py --codes-file low_priced_codes.txt \
    --from 2023-01-01 --to 2026-03-31 --sleep 2.0
#   → data/raw に保存。以降は「保存済みデータ」で何度でも高速に検証できる

# 2. バックテスト(CLI)
python scripts/run_backtest.py --codes 7203 6758 9984
#   データ未取得でも合成データで動作確認:
python scripts/run_backtest.py --demo

# 3. ダッシュボード(ブラウザ・PC/iPhone対応)
streamlit run dashboard/app.py

# 3'. スマホ向け静的HTMLレポート(サーバ不要・すぐ見られる)
python scripts/run_backtest.py --demo   # → output/report.html
```

## 📈 検証結果(暫定・2026-07)

低位・高ボラ株の実データ(J-Quants V2 日足)で「5日線押し目タッチ・リバウンド」を検証。

### 分かったこと
- **手法のロジックには根拠がある**: エッジ分析で「5日線からの乖離が大きいほど反発しや
  すい」というシンジ氏の核心主張を支持する傾向を確認(乖離3〜4%台が最良)。
  ただし乖離**5.6%超は逆にマイナス**(落ちるナイフ)= 上限も必要。
- **乖離帯を 3〜5% に絞ると**、調整に使った銘柄では期待値 +0.8%/トレード・
  **t値3.01(統計的に有意)**・保守PF2.44・単純保有(−32%)を圧倒。
- **別銘柄(アウト・オブ・サンプル)でも方向は一致**(期待値+0.45%・保守PF1.51・
  Buy&Hold圧勝)し、過学習での崩壊は見られなかった。

### 正直な限界(まだ「優位性あり」と断定はできない)
- **サンプルが小さすぎる**: 無料プランのレート制限で取得銘柄が数銘柄に限られ、
  OOS検証は**トレード12件**しか集まらず **t=0.69(ノイズと区別できない)**。
- パラメータ(乖離帯・損切り・利確)を**同一データで調整**しており、
  カーブフィットの可能性が残る。
- **1日100回のスキャルは日足では再現不可**。板読み・特別気配回避も対象外。
  シンジ氏本来の優位性は日中の高頻度スキャルにあり、日足検証はその**一部
  (押し目の方向性エッジ)**のみを見ている。

### 決着に必要なこと
1. **サンプルを桁で増やす**: 多数銘柄×長期間。→ ローカルPCでゆっくり取得(下記)
   か、レート枠回復後に小分け取得。
2. **分足での忠実検証**: J-Quants の分足/Tickアドオン(Lightプラン+¥5,500/月、
   `/equities/bars/minute`)で、ザラ場のタッチ→数百円利確を再現。
   ※ リアルタイム売買には kabuステーションAPI 等の証券会社APIが別途必要。
