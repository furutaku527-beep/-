# 検証対象手法 — 「5日線」を軸にしたデイトレード

出典: シンジ氏 note「【3つ目の気付き】僕の1番の武器で生命線」(スキャルピングノート)
本ファイルは note 本文を**ルールベース(機械判定可能)な条件**に整理したもの。
原文は裁量スキャルピング(1日100〜500約定・板読みベース)であり、機械化に際して
解釈・近似した箇所を明記する。

---

## 1. 手法の核心(原文の要旨)

- **5日移動平均線(日足・以下「5日線」)が最重要ライン。次点で25日線。**
  株価は5日線に「吸い寄せられる」動きをし、最も信頼度の高いサポート/レジスタンスになる。
- **基本姿勢: 5日線の「上」でトレード(買い)、5日線の「下」ではノートレード(静観)。**
  - 株価 > 5日線 … 5日線はサポート(押し目買いの土俵)
  - 株価 < 5日線 … 5日線はレジスタンス(上ヒゲになりやすく期待値が低い)
- **5日線からの「乖離」がリスク指標。**
  - 5日線から離れた高い位置 = 危険ゾーン(連売り・特別売気配に巻き込まれやすい)
    → エントリーしても**OUT(手仕舞い)を早く**、ロットを抑える。
  - 5日線に近い位置でのエントリー = 安全度が高く、保有時間を長めに取れる。
- **移動平均線の「向き」を見る。** 5日線・25日線が上向きか下向きか。
- **急落幅が大きいほど5日線・25日線での反発確率が高い。**

### 最重要シグナル(原文「信頼度が高いパターン」=激アツ)
> 5日線・25日線から**乖離した位置からの、5日線タッチ / 25日線タッチ**。
> 前日終値とラインの値幅が大きいほど期待値が高い。
> 「高い位置からの急降下でのタッチ」がベスト("高い所から落としたボールほど跳ねる")。

### 損切り(原文)
- 「**最安値を更新したら損切**」(直近安値ラインを割れたら撤退)。
- 5日線は「最後の砦」。5日線が下にある限りは耐える根拠になるが、
  **割れたら必ず損切**(「絶対安全なポイントは存在しない」)。

### 除外条件(トレードしない/監視から外す)
- 5日線の**下**で株価推移 → 監視から削除(レジスタンス化しやすい)。
- 5日線が**意識されていない**銘柄。
- **東証プライム(旧東証1部)** … オーバーシュートしやすく苦手 → 弱気/除外寄り。
- **決算・悪材料が出た銘柄**、**指数の大暴落日** → 5日線の考えは適用外。
- **IPO直後の銘柄**(イレギュラーな動き) → 除外。
- 5日線が近くても**25日線が大きく乖離**している時は注意。

### 例外(下降トレンドの逆張り big chance)
- 上昇トレンドで5日線から乖離 = 危険。だが**下降トレンドで5日線から乖離しすぎた位置は
  逆にチャンス**(売られすぎの反発狙い、例: ストップ安が5日線の位置)。
- ただし条件が厳しい:
  - 始値とストップ安の値幅が**小さい**と張り付きリスク → スルー。
  - 25日線からの乖離が**大きすぎる**と売り圧力が強く危険。
  - 決算下げ・悪材料下げ・指数暴落での逆張り(スケベ買い)は厳禁。

## 2. 日足で検証できること / 分足が必要なこと

原文は本質的に**ザラ場の板読み + 分単位のスキャル**。J-Quants 無料プランは
**日足のみ**(分足なし)のため、検証範囲を分けて扱う。

| 項目 | 日足で可能 | 分足が必要 |
|---|---|---|
| 5日線/25日線の押し目・反発の**期待値**(終値ベース) | ✅ | |
| 乖離度とリスクの関係(乖離大→翌日下落 等) | ✅ | |
| 「5日線の上だけトレード」の優位性 | ✅ | |
| 同一日内の**正確なエントリー/エグジット価格**(タッチの瞬間) | △(OHLCで近似) | ✅ |
| 板読み・特別売気配の回避・連売り検知 | ❌ | ✅(板データは無料外) |
| 1日100〜500回のスキャルの再現 | ❌ | ✅ |

→ **本システムはまず「日足ベースの押し目タッチ・リバウンド戦略」を検証する。**
日中の値動きは日足 OHLC で近似し、近似の限界は結果に明記する。
分足が取得できる環境(有料プラン等)になれば、より忠実な日中検証へ拡張可能な設計とする。

## 3. ルールベース実装(日足版・パラメータ化)

ロング(買い)・同一日内デイトレード。各パラメータは設定で変更可能(ダッシュボードで調整)。

### 3-1. 銘柄スクリーニング(母集団 = 低位・高ボラ株)
> ⚠️ **重要**: 本手法は大型株ではなく**低位・小型の値動きが大きい株**が対象。
> 著者本人が note で「**東証1部(プライム)は苦手**、オーバーシュートしやすい」と
> 明言している。よって母集団は**全銘柄から自動スクリーニング**して作る
> (`src/data/screener.py`)。

スクリーニング条件(基準日の全銘柄スナップショットから抽出):
- **低位**: 終値 < `max_price`(既定 **1000円**)。
- **流動性**: 売買代金(終値×出来高) ≥ `min_turnover`(既定 5千万円)。
- (任意)**日中変動率** ≥ `min_range_pct`(基準日の (高値−安値)/終値)。
- (任意)**プライム市場を除外** `exclude_prime`(既定 true、上場マスタからベストエフォート)。
- 売買代金の大きい順に上位 `top_n` 銘柄を採用。
- さらに各銘柄で上場後 `min_history_days`(60営業日)以上を要求(IPO直後を除外)。

### 3-2. メイン戦略: 5日線 押し目タッチ・リバウンド(LONG)
> ⚠️ **修正(手法の忠実化)**: 当初は「3%以上の乖離」を必須にしていたが、note本文の
> 基本は「**5日線から近い位置でトレード**」「**5日線の上で買い**」。乖離は
> 「リスク指標 / 期待値ブースター」であって必須ゲートではない。よって `dev_min` の
> 既定を **0** にし、上昇中の5日線の上で押し目タッチすればエントリーする。
> `dev_min` を上げると「大きな乖離からの押し目(激アツ)」だけに絞れる。

当日 t のシグナル(**先読み防止**: 判定は前日までの確定データのみ。基準線 `line` =
前日までで確定した5日線):
1. **上昇地合い**: `MA5[t-1] > MA5[t-2]`(前日時点で5日線が上向き)。
2. **5日線の上**: `Close[t-1] > MA5[t-1]`(前日は線の上)。
3. **乖離(任意)**: `dev5[t-1] ≥ dev_min`(既定0 = 実質無効。`dev5` は分析軸として記録)。
4. **当日に5日線へ押し目タッチ**: `Low[t] ≤ line`(日中に前日5日線まで下落)。
5. (任意)寄り付きが線付近以上: `Open[t] ≥ line*(1 - open_buf)`。

**トレード(未来情報を使わない / 同一日内で完結):**
- エントリー: 押し目タッチ価格 ≒ `line`(前日確定の5日線でリバウンドを拾う)+ スリッページ。
- 損切り: `Low[t] < line*(1 - stop_pct)`(既定 1.5%)なら `line*(1 - stop_pct)` で損切。
- 利確/手仕舞い: 損切に掛からなければ **当日終値 `Close[t]` で決済**(デイトレ完結)。
- 1トレード損益 = 決済 − エントリー − 手数料 − スリッページ。

> **「1日100回」について**: 著者は分単位の板読みスキャル。日足では1銘柄1日1回が上限の
> ため、その回数の再現は不可能。代わりに**低位株を多数スクリーニングして横断適用**し、
> 「市場全体で毎日多数のトレードが発生する」状態で**エッジの有無**を検証する。

### 3-3. 例外戦略: 急落乖離リバウンド(任意・実験的, 既定OFF)
下降局面での売られすぎ反発狙い。原文どおりリスクが高く、決算/指数暴落フィルタ必須。
既定では無効。`enable_oversold_rebound=true` で有効化し、別途検証する。

### 3-4. グローバル除外(全戦略共通)
- (任意)決算発表日 ±1日を除外(J-Quants の決算データが取得できる場合)。
- (任意)指数(TOPIX/日経)が当日 `index_crash_pct`(既定 −3%)以上下落した日を除外。

## 4. パラメータ既定値(まとめ)

| パラメータ | 既定 | 意味 |
|---|---|---|
| `ma_fast` | 5 | 5日線 |
| `ma_slow` | 25 | 25日線 |
| `dev_min` | 0.0 | エントリーに必要な最小乖離(0=実質無効。上げると激アツ限定) |
| `stop_pct` | 0.015 | 5日線割れ損切り幅 |
| `open_buf` | 0.01 | 寄り付き許容バッファ |
| `min_history_days` | 60 | IPO除外 |
| `min_turnover` | 5e7 | 最小売買代金(円) |
| `max_price`(screener) | 1000 | 低位株の上限株価 |
| `exclude_prime`(screener) | true | プライム市場除外 |
| `fee_rate` | 0.0005 | 片道手数料率(概算) |
| `slippage_pct` | 0.001 | 片道スリッページ |

これらは `src/strategy/params.py` の `StrategyParams` で一元管理する。
